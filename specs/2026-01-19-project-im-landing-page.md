# Project_IM Landing Page Spec

## Problem statement and context
需要一个用于发布 Project_IM 的 landing page，突出“轻量 + 高隐私”的定位，并支持中英双语展示。用户提供手写风画稿作为视觉参考，无现成 logo。

## Goals
- 清晰表达产品定位与核心价值，降低首次访问理解成本。
- 提供可下载入口与联系开发者通道。
- 完成中英双语版本，页面加载在桌面与移动端均正常。

## Non-goals
- 不包含用户登录、账户系统或聊天功能演示。
- 不做复杂的 CMS 或多页面站点。

## Requirements and constraints

### Functional
- 单页 landing page，包含下载按钮与联系开发者入口（邮箱）。
- 支持中英双语切换或同页双语展示（需二选一并保持一致）。
- 可替换的截图/插画占位区，用于未来放产品截图或手绘插画。

### UX / flows
- Hero 区域展示产品名、主标语、简短副文案与主要 CTA。
- 明确的价值点列表（3-5 条），聚焦轻量与隐私。
- 联系开发者区域展示邮箱：`xiaoyuchen031204@gmail.com`。
- 视觉风格遵循“手写风/画稿感”，整体亲和但清晰易读。

### Performance
- 资源尽量轻量，避免大体积脚本或多余依赖。

### Security / privacy
- 页面不采集用户数据，不嵌入第三方追踪脚本。

## User journeys / flows
- 访问者进入页面 -> 看到产品定位与主要 CTA -> 点击下载。
- 访问者了解功能亮点 -> 需要咨询 -> 通过邮箱联系开发者。

## Acceptance criteria
- 页面为单页且包含 Hero、价值点、下载 CTA、联系开发者区域。
- 中英双语可用且无明显排版冲突。
- 手写风视觉方向明确（字体、配色、线条/插画风格统一）。
- 点击“下载”按钮有可配置链接占位。
- 邮箱显示为可点击 mailto 链接。

## Open questions

- 是否已有手写风画稿可供直接使用？
